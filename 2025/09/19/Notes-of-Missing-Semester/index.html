什么是Shell？

Shell 是操作系统与用户之间的命令解释器，它充当用户与内核（操作系统的核心）之间的交互接口，让用户能够通过输入命令来操作计算机。

简单来说，当你在终端（Terminal）中输入命令（如 ls、cd、cp 等）时，正是 Shell 负责解析这些命令，并将其转化为内核能理解的指令，然后将执行结果返回给用户。

在Shell中导航

shell 中的路径是一组被分割的目录，在 Linux 和 macOS 上使用 / （斜杠）分割，而在 Windows 上是 \（反斜杠）。路径 / 代表的是系统的根目录，所有的文件夹都包括在这个路径之下，在 Windows 上每个盘都有一个根目录（例如： C:\）。 如果某个路径以 / 开头，那么它是一个 绝对路径，其他的都是 相对路径 。相对路径是指相对于当前工作目录的路径，当前工作目录可以使用 pwd 命令来获取。此外，切换目录需要使用 cd 命令。在路径中，. 表示的是当前目录，而 .. 表示上级目录：

超级用户

对于大多数的类 Unix 系统，有一类用户是非常特殊的，那就是：根用户（root user）。 你应该已经注意到了，在上面的输出结果中，根用户几乎不受任何限制，他可以创建、读取、更新和删除系统中的任何文件。 通常在我们并不会以根用户的身份直接登录系统，因为这样可能会因为某些错误的操作而破坏系统。 取而代之的是我们会在需要的时候使用 sudo 命令。顾名思义，它的作用是让您可以以 su（super user 或 root 的简写）的身份执行一些操作。 当您遇到拒绝访问（permission denied）的错误时，通常是因为此时您必须是根用户才能操作。然而，请再次确认您是真的要执行此操作。

有一件事情是你必须作为根用户才能做的，那就是向 sysfs 文件写入内容。系统被挂载在 /sys 下，sysfs 文件则暴露了一些内核（kernel）参数。 因此，不需要借助任何专用的工具，就可以轻松地在运行期间配置系统内核。
